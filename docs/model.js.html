  • ¶

    A Model is an abstraction over a business model, a data container with logic. It provides:

    • attribute setters and getters
    • validations
    • serialization as JSON
    • events

    It’s expected that this class will be extended.

    It’s a good idea to listen to changes in the model and trigger UI events based on those changes.

    See LoudAccessors: https://github.com/lucaong/loud-accessors See validators: lib/assets/javascripts/validators.js.coffee

    Example:

    class Product extends Lib.Model @attrAccessible ‘title’, ‘description’, ‘price’

    @validatesPresenceOf 'title'
@validatesPresenceOf 'price'

formattedPrice: -> '$' + @get('price')

    product = new Product(title: ‘Dress’, price: 21.99) product.on ‘change:price’, (event, attribute, value) -> alert(“New price! #{value}”)

    Standard Model extends from Module which has extend and include similar to rails

      class Model extends Lib.Module
